## Visão Geral

Pause temporariamente as cobranças de uma assinatura. O cliente pode manter ou perder acesso durante a pausa, dependendo da sua implementação. Use o endpoint de [retomar](/api-reference/assinaturas/retomar) para reativar as cobranças.

## Exemplo de Requisição

<CodeGroup>

```bash cURL
curl -X POST https://api.garu.com.br/api/subscriptions/100/pause \
  -H "Authorization: Bearer sk_test_sua_chave"
```

```javascript JavaScript
const response = await fetch(
  'https://api.garu.com.br/api/subscriptions/100/pause',
  {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${process.env.GARU_API_KEY}`
    }
  }
);

const result = await response.json();
console.log(`Status: ${result.status}`); // "paused"
console.log(`Pausada em: ${result.pausedAt}`);
```

```python Python
import requests
import os

url = "https://api.garu.com.br/api/subscriptions/100/pause"
headers = {
    "Authorization": f"Bearer {os.environ['GARU_API_KEY']}"
}

response = requests.post(url, headers=headers)
result = response.json()
print(f"Status: {result['status']}")  # "paused"
print(f"Pausada em: {result['pausedAt']}")
```

</CodeGroup>

## Resultado

- Status muda para `paused`
- Cobranças são suspensas
- Nenhum pagamento será tentado enquanto pausada
- Use [retomar](/api-reference/assinaturas/retomar) para reativar

## Resposta de Sucesso (200 OK)

```json
{
  "id": 100,
  "status": "paused",
  "pausedAt": "2024-02-20T10:00:00.000Z",
  "message": "Assinatura pausada com sucesso"
}
```

## Considerações

<Note>
  **Acesso durante a pausa**: A decisão de manter ou revogar o acesso do cliente durante a pausa é da sua aplicação. A Garu apenas suspende as cobranças.
</Note>

<Warning>
  Assinaturas pausadas por muito tempo podem precisar de atualização do método de pagamento ao serem retomadas.
</Warning>

## Erros Comuns

<AccordionGroup>
  <Accordion title="404 - Assinatura não encontrada">
    ```json
    {
      "statusCode": 404,
      "message": "Subscription not found"
    }
    ```
    **Solução:** Verifique se o ID da assinatura está correto.
  </Accordion>

  <Accordion title="400 - Assinatura não pode ser pausada">
    ```json
    {
      "statusCode": 400,
      "message": "Cannot pause subscription with current status"
    }
    ```
    **Solução:** Apenas assinaturas ativas podem ser pausadas.
  </Accordion>
</AccordionGroup>
