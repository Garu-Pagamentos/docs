## Visão Geral

Exclua um preço de assinatura. Preços com assinaturas ativas não podem ser excluídos.

<Warning>
  Não é possível excluir preços com assinaturas ativas. Desative o preço (`isActive: false`) usando o endpoint de [atualização](/api-reference/assinaturas/precos/atualizar) em vez de excluir.
</Warning>

## Exemplo de Requisição

<CodeGroup>

```bash cURL
curl -X DELETE https://api.garu.com.br/api/subscription-prices/456 \
  -H "Authorization: Bearer sk_test_sua_chave"
```

```javascript JavaScript
const response = await fetch(
  'https://api.garu.com.br/api/subscription-prices/456',
  {
    method: 'DELETE',
    headers: {
      'Authorization': `Bearer ${process.env.GARU_API_KEY}`
    }
  }
);

if (response.status === 204) {
  console.log('Preço excluído com sucesso');
}
```

```python Python
import requests
import os

url = "https://api.garu.com.br/api/subscription-prices/456"
headers = {
    "Authorization": f"Bearer {os.environ['GARU_API_KEY']}"
}

response = requests.delete(url, headers=headers)

if response.status_code == 204:
    print("Preço excluído com sucesso")
```

</CodeGroup>

## Resposta de Sucesso (204 No Content)

A resposta não contém corpo quando a exclusão é bem-sucedida.

## Quando Usar

- O preço foi criado por engano
- O plano nunca foi utilizado
- Não há assinaturas ativas neste preço

## Alternativa: Desativar

Se o preço tem assinaturas ativas, desative-o em vez de excluir:

```bash
curl -X PATCH https://api.garu.com.br/api/subscription-prices/456 \
  -H "Authorization: Bearer sk_test_sua_chave" \
  -H "Content-Type: application/json" \
  -d '{"isActive": false}'
```

Isso impede novas assinaturas sem afetar as existentes.

## Erros Comuns

<AccordionGroup>
  <Accordion title="400 - Preço possui assinaturas ativas">
    ```json
    {
      "statusCode": 400,
      "message": "Cannot delete price with active subscriptions"
    }
    ```
    **Solução:** Desative o preço em vez de excluí-lo, ou cancele todas as assinaturas primeiro.
  </Accordion>

  <Accordion title="404 - Preço não encontrado">
    ```json
    {
      "statusCode": 404,
      "message": "Subscription price not found"
    }
    ```
    **Solução:** Verifique se o ID do preço está correto.
  </Accordion>
</AccordionGroup>
